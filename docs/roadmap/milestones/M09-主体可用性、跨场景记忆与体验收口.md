# M09：主体可用性、跨表面连续性与体验收口

- 状态：completed
- 完成日期：2026-07-14
- 当前事实源：[Cognition 当前视图](../../architecture/current/07-子系统当前视图/Cognition.md)、[Extension 与 Skill Plane 当前视图](../../architecture/current/07-子系统当前视图/Extension与SkillPlane.md)、[ADR-0008](../../architecture/decisions/ADR-0008-ExperienceLedger与版本化Memory.md)

## 目标成果

让当前角色在本地 Desktop 与 Extension 提供的外部表面上保持可追溯、可隔离、可长期运行的连续性，同时让人物 Skill 调用、Control Center 状态和失败恢复进入真实可用链路。

## 完成结果

- 外部 Adapter 只提交 `ConversationAddress` 和清洗后的 perception/evidence proposal；Kernel 生成 canonical `ConversationContext`、不可逆 actor id 与 scope。
- Experience Ledger 成为唯一经历事实源；Conversation Store、Episode 和关系统计都是可重建投影。
- Desktop 长期对话不依赖用户手动新建会话；Conversation 自动形成 Chapter 与 Segment，并从持久投影恢复有界 Working Set。
- Context Assembly 在检索前执行 scope 与 owner 过滤，再装配 Conversation State、近期消息、历史 Segment、Memory、Knowledge、Relationship 和 Experience。
- 长期记忆由运行中的 `MaintenanceScheduler` 消费持久 consolidation job；停机只 flush、checkpoint、封口和入队，不执行模型推理。
- Extension 不拥有角色会话、经历或 Memory CRUD；管理能力和 adapter 能力不会进入人物 Skill catalog。
- Core Skill 的 catalog、规划、Policy、确认、Gateway、handler、审计和 Cognition synthesis 已形成真实闭环。
- Control Center 的记忆预览区分 Conversation 消息、Experience Moment、Memory revision 与角色知识，不把预览冒充实际 Prompt 命中。

## 删除的旧主线

- Cognition 场景短期记忆与 scene-session 双写。
- Kernel 持有的扩展平行会话数据表。
- Extension 自建的平行会话事实入口与权限。
- Renderer 对话历史事实源。
- 跨权限域的“相关就召回”行为。
- 停机阶段执行长期记忆模型推理。

## 验收证据

- Protocol Schema 与 TS/Python 生成物统一。
- Cognition 覆盖 Ledger 重建、Conversation 重建、scope 过滤、Episode、持久任务、证据校验与运行中巩固。
- Kernel 覆盖 Conversation Directory 稳定性、地址隐私、Extension 权限、evidence cognitive effect、Skill 与 lifecycle。
- Desktop 类型检查覆盖新的 Conversation/Experience/Memory 预览模型。
