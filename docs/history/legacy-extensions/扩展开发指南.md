# 扩展开发指南

> SDK：`@glimmer-cradle/extension-sdk`
> 当前事实源：`docs/architecture/current/05-插件层.md`

## 1. 设计心智

Extension 是月见的外部生态边界。它不是固定的 adapter/tool/ui/skill 分类，而是这些维度的自由组合：

- `contributes`：我给月见贡献什么。
- `requires`：我需要宿主开放哪些端口。
- `permissions`：我被授权做什么。
- `lifecycle`：我何时激活、何时释放资源。

WebSocket、HTTP、stdio、MCP 等连接方式只是实现细节。需要 WebSocket 时使用 `WebSocketBridge`，不要把连接方式当成扩展类型。

## 2. 目录结构

```text
extensions/<bucket>/<extension-id>/
├── extension-manifest.yaml
├── package.json
├── tsconfig.json
├── index.ts
├── config/
│   └── schema.ts
└── src/
    └── <extension-id>-extension.ts
```

## 3. Manifest

```yaml
id: hello-extension
name: Hello Extension
version: 0.1.0
description: My first Selrena extension.
main: dist/index.js
minAppVersion: 0.1.0

engines:
  selrena: 0.1.0

permissions:
  - CONFIG_READ_SELF
  - COMMAND_REGISTER
  - AGENT_REGISTER

requires:
  - commands
  - agents

activationEvents:
  - onStartup
  - onCommand:hello-extension.say

contributes:
  commands:
    - command: hello-extension.say
      title: Say Hello
      category: Hello
  skills:
    - id: hello-skill
      name: 问候技能
      description: 提供一个可被月见调用的简单问候动作。
      tools:
        - name: say_hello
          description: 生成一句问候。
          parameters:
            type: object
            properties:
              name:
                type: string
      policy:
        riskLevel: low
        confirmationRequired: false
        sideEffects: []
        audit: true
```

`requires` 是端口意图，`permissions` 是实际授权。调用受保护 API 时，Kernel 只看权限。`contributes.skills` 是声明式目录项，加载扩展时会进入 `SkillCatalogSnapshot`；真正可执行的 handler 仍要在激活后通过 `ctx.ports.agents.registerSubAgent(...)` 绑定同一个本地 `id`。

## 4. 入口文件

```ts
import { defineExtension } from '@glimmer-cradle/extension-sdk';
import { HelloExtension } from './src/hello-extension';

export default defineExtension({
  manifest: {
    engines: {
      selrena: '0.1.0',
    },
    requires: ['commands', 'agents'],
    activationEvents: ['onStartup'],
    contributes: {
      commands: [
        {
          command: 'hello-extension.say',
          title: 'Say Hello',
          category: 'Hello',
        },
      ],
      skills: [
        {
          id: 'hello-skill',
          name: '问候技能',
          description: '提供一个可被月见调用的简单问候动作。',
          tools: [
            {
              name: 'say_hello',
              description: '生成一句问候。',
              parameters: {
                type: 'object',
                properties: {
                  name: { type: 'string' },
                },
              },
            },
          ],
        },
      ],
    },
  },
  extension: new HelloExtension(),
});
```

`index.ts` 只导出定义，不写业务逻辑。

## 5. 扩展类

```ts
import { BaseExtension } from '@glimmer-cradle/extension-sdk';
import { HelloConfigSchema, type HelloConfig } from '../config/schema';

export class HelloExtension extends BaseExtension<HelloConfig> {
  constructor() {
    super(HelloConfigSchema);
  }

  protected override activate(): void {
    this.registerCommand(
      'hello-extension.say',
      async (target?: unknown) => {
        const name = typeof target === 'string' ? target : 'world';
        return `${this.config.greeting}, ${name}!`;
      },
      { title: 'Say Hello', category: 'Hello' },
    );
  }
}
```

## 6. Host Ports

扩展通过 `ctx.ports` 使用宿主开放能力：

```ts
this.ctx.ports.perception.inject(event);
this.ctx.ports.sceneAttention.reportSceneAttention(sceneId, true);
this.ctx.ports.shortTermMemory.append(entry);
this.ctx.ports.events.emit(`extension.${this.ctx.extensionId}.ready`, {});
this.ctx.ports.agents.registerSubAgent(profile);
```

不要直接 import Kernel 内部模块，也不要把平台原始字段传给 Cognition。

## 7. WebSocket 接入

```ts
import { BaseExtension } from '@glimmer-cradle/extension-sdk';
import { WebSocketBridge } from '@glimmer-cradle/extension-sdk/utilities/websocket';

export class PlatformExtension extends BaseExtension {
  private bridge: WebSocketBridge | null = null;

  protected override activate(): void {
    this.bridge = new WebSocketBridge(this.logger, {
      onJsonMessage: (data) => this.handleMessage(data),
    });
    this.addDisposable(this.bridge);
    this.bridge.start({ host: '127.0.0.1', port: 8080 });
  }

  private handleMessage(data: unknown): void {
    // 在这里清洗平台原始消息，再注入标准 PerceptionEvent。
  }
}
```
