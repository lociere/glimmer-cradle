# Electron 启动失败：ELECTRON_RUN_AS_NODE 环境变量污染

> 历史事故记录：当前 Electron/Desktop 开发入口见 [桌面与 Avatar 开发](../../guides/subsystems/桌面与Avatar开发.md)。

**日期**：2026-05-08  
**影响**：`pnpm dev` 启动时 Electron 主进程立即崩溃，桌宠窗口无法打开。

## 症状

`pnpm dev` 后 Electron 部分立即退出，错误日志指向 `electron_1.app.whenReady()`，但 `electron_1.app` 是 `undefined`。调用栈中出现 `node:electron/js2c/node_init`，说明 Electron 二进制被启动了，但运行模式异常。

## 排查

用探针脚本检查 `require('electron')` 返回值：

```js
const e = require('electron');
console.log('typeof:', typeof e);
console.log('value:', typeof e === 'string' ? e : Object.keys(e).slice(0, 10));
```

Electron CLI 返回的是字符串路径而不是 API 对象。继续检查环境变量：

```powershell
Get-ChildItem env: | Where-Object { $_.Name -match 'ELECTRON' }
```

发现 `ELECTRON_RUN_AS_NODE=1`。

## 根因

全局环境变量 `ELECTRON_RUN_AS_NODE=1` 强制 Electron 二进制以 Node.js 模式启动。此时 `require('electron')` 返回 electron.exe 路径字符串，不返回 Electron API 对象，因此 `app.whenReady()` 为空。

## 修复

在桌面 dev launcher 中显式删除该变量，并用专用 Node.js 脚本启动 Electron：

```js
delete process.env.ELECTRON_RUN_AS_NODE;
process.env.VITE_DEV = '1';
```

同时 dev launcher 等待 Vite HTML 入口可访问后再启动 Electron，避免 Electron 与 Vite 抢跑。

## 教训

- `require('electron')` 返回字符串时，优先检查 `ELECTRON_RUN_AS_NODE`。
- cross-env 只能 set，不能可靠 unset；需要在 Node/PowerShell/bash 中显式删除环境变量。
- Electron dev launcher 应等待 Vite 实际入口可服务。
- Kernel 未 ready 不等于 UI 启动失败，UI 应显示等待态。
