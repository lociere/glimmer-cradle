# 内核启动失败：desktop-ui-controller 的 .js 后缀引入不一致

> 历史事故记录：当前 Kernel 开发入口见 [Kernel 开发](../../guides/subsystems/Kernel开发.md)。

**日期**：2026-05-08  
**影响**：`pnpm dev` 启动时 Kernel 入口崩溃，Cognition、Avatar 等模块无法初始化。

## 症状

Kernel 端报错：

```text
Error: Cannot find module '../../../foundation/event-bus/event-bus.js'
Require stack:
- core/kernel/src/core/application/capabilities/desktop-ui/desktop-ui-controller.ts
- core/kernel/src/app.ts
- core/kernel/src/main.ts
```

## 排查

`desktop-ui-controller.ts` 的相对 import 带 `.js` 后缀，但项目其余 Kernel TS 文件使用无后缀风格。dev 模式通过 `ts-node-dev --transpile-only` 走 CommonJS 解析：

- `require('./foo')` 可以解析 `.ts`；
- `require('./foo.js')` 会严格找 `.js`，不会 fallback 到 `.ts`。

因此构建可能通过，但 dev runtime 会找不到模块。

## 根因

单个文件采用了 Node16 ESM 风格 `.js` 后缀，和项目当前 dev runtime 的 CommonJS 解析风格不一致。

## 修复

去掉该文件相对 import 的 `.js` 后缀，使其与当前 Kernel 代码风格一致。

## 教训

- 改导入风格前先 `rg` 当前项目约定。
- tsconfig 的 Node16 设置不等于 dev runtime 已经切到 ESM。
- 若未来迁移 ESM，必须整体切换 dev runner 和所有相对 import 风格，不能只改一个文件。
