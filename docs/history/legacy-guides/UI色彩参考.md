# 月见 UI 色彩参考

> 本文定义 Control Center 当前唯一有效的色彩基线：标准 Light 与 Dark 主题。人物设定色、动态取色和第三方主题不属于当前实现范围。

## 设计立场

Control Center 首先是桌面生产力软件。基础主题必须在没有角色色和装饰材质时仍然自然、清楚、耐看。因此当前只保留三类颜色：

1. **中性表面**：建立窗口、导航、工作区、输入和浮层的明暗层级。
2. **系统蓝**：用于选中、焦点、链接和当前作用域唯一的主要动作。
3. **语义状态**：成功、警告和错误独立表达，不受主题强调色影响。

不使用青色铺底、金色品牌背景、彩色渐变或根据人物设定染色整个界面。

## Light

| Token | 参考值 | 用途 |
|---|---|---|
| `background` | `#f3f3f3` | 窗口与主工作区底色 |
| `background-elevated` | `#fafafa` | 标题栏和轻提升区域 |
| `surface` | `#ffffff` | 输入、独立内容和浮出对象 |
| `surface-subtle` | `#f7f7f7` | 列表分组与次级区域 |
| `surface-hover` | `#ebebeb` | hover 与轻交互反馈 |
| `text-primary` | `#1f1f1f` | 标题与正文 |
| `text-secondary` | `#616161` | 辅助说明 |
| `outline` | `#e1e1e1` | 必要的低对比分界 |
| `accent` | `#0f6cbd` | 主要动作、选中与焦点 |

Light 不是纯白画布上堆灰色描边。大区域通过接近白色的表面变化、留白和对齐形成边界；描边只保留给输入、连续列表和需要明确命中的对象。

## Dark

| Token | 参考值 | 用途 |
|---|---|---|
| `background` | `#1e1e1e` | 窗口与主工作区底色 |
| `background-elevated` | `#252526` | 标题栏和轻提升区域 |
| `surface` | `#252526` | 输入、独立内容和浮出对象 |
| `surface-subtle` | `#2d2d30` | 列表分组与次级区域 |
| `surface-hover` | `#343434` | hover 与轻交互反馈 |
| `text-primary` | `#f3f3f3` | 标题与正文 |
| `text-secondary` | `#c5c5c5` | 辅助说明 |
| `outline` | `#3f3f46` | 必要的低对比分界 |
| `accent` | `#479ef5` | 主要动作、选中与焦点 |

Dark 使用中性炭黑而不是蓝黑、青黑或纯黑。不同表面之间只保留足以感知层级的明度差，避免每个模块都变成发亮卡片。

## 语义状态

- Success 使用绿色，表示真正可用或操作成功。
- Warning 使用琥珀色，表示可继续但需要注意。
- Error 使用红色，表示失败、阻塞或危险动作。
- 状态必须同时包含文字或图标，不能只依赖颜色。

## 材质边界

- Mica 只允许出现在受支持平台的窗口根层或导航底层，失败时退化为基础主题色。
- Acrylic 只用于菜单、对话框和临时浮层。
- 主工作区、长文本、表单和状态列表保持高不透明度。
- 材质不能改变 Light / Dark 的基础 token，也不能成为第三套主题。

## 后续扩展条件

只有当 Light 与 Dark 已通过视觉回归、键盘操作、高对比度和 Windows 125% / 150% 缩放验证后，才讨论角色主题、用户强调色或 MD3 动态取色。扩展主题必须建立在相同语义 token 上，不能重新定义控件结构。
