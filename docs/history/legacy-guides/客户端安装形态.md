# 客户端安装形态

本文定义月见桌面客户端安装完成后的目标形态。它描述部署边界，不复制源码目录，也不记录当前迁移进度；构建方法见 [客户端打包指南](./客户端打包指南.md)。

## 设计原则

1. **安装资源只读，用户数据可写**：升级可以替换应用，不破坏月见的连续性状态。
2. **源码按领域组织，安装包按部署单元组织**：两棵目录树服务不同问题，不要求一一同名。
3. **组件由 Kernel 监督**：独立进程都有入口、readiness、健康、停机与版本契约。
4. **路径由清单解析**：不依赖仓库路径、开发机绝对路径或角色文件名。
5. **供应商依赖可追溯**：第三方运行时、许可与完整性摘要必须可审计。

## Windows 安装目录

```text
Glimmer Cradle/
├── Selrena.exe                         # Desktop Shell 与应用入口
└── resources/
    ├── app.asar                        # Electron main / preload / Control Center
    ├── components/
    │   ├── kernel/                     # 全局监督、编排与协议中枢
    │   ├── cognition/                  # Cognition 认知核
    │   ├── avatar-shell/
    │   │   ├── host/                   # Composition Host 平台实现
    │   │   └── unity-worker/           # Unity / Cubism 无窗口 Render Worker
    │   └── engines/
    │       ├── audio/                  # ASR / TTS Engine
    │       ├── vision/                 # 可选视觉 Engine
    │       └── vector/                 # 可选向量 Engine
    ├── native/
    │   └── <platform-arch>/            # 可复用原生库与平台底座
    ├── assets/                         # 随版本发布的只读产品资产
    ├── defaults/                       # 无密钥默认配置与 Schema
    └── component-manifest.json         # 部署入口、依赖、版本与完整性事实源
```

`components/` 中的并列表示它们是可独立部署、监督和更新的运行单元，不表示逻辑架构同层。Kernel 仍是中枢；Cognition 是心智；Avatar Shell 是身体；Engine 是能力执行器。

Desktop 不再作为 `components/desktop/` 重复出现。`Selrena.exe` 与 Electron 约定位置的 `resources/app.asar` 共同构成 Desktop Shell，负责托盘、Control Center、启动交接和用户级退出请求。

## 运行角色

```text
Selrena.exe
    ↓ 启动并连接
Kernel
    ├── Cognition
    ├── Engines
    └── Avatar Shell
          ├── Composition Host
          └── Unity Render Worker
```

- Desktop Shell 提交“退出月见”，不直接粗暴遗留子进程。
- Kernel 维护监督树并按逆序停机。
- Composition Host 拥有 Avatar Shell 唯一可见透明窗口。
- Unity Worker 不拥有正式桌面窗口，只产生带 alpha 的身体画面。
- Composition Host 在 Worker 超时后负责回收其进程，Kernel 再回收整个 Avatar Shell 组件。

## Avatar Shell 安装边界

### Composition Host

`components/avatar-shell/host/` 是应用级宿主，负责：

- DirectComposition 或目标平台等价组合 API；
- 共享 GPU surface 的挂接与提交；
- 透明窗口、DPI、显示器、停靠和位置恢复；
- 身体命中、透明区穿透、拖动与点击；
- `surface-ready`、`visible`、`stopped` 生命周期。

它不读取 Cubism 模型参数，不决定表情含义，也不调用 Cognition。

### Unity Render Worker

`components/avatar-shell/unity-worker/` 负责：

- Cubism4 / Cubism5 模型生命周期；
- 动作、表情、视线、口型和参数驱动；
- 将带预乘 alpha 的帧写入共享 GPU surface；
- 上报模型与渲染 readiness。

发布模式不得显示 Unity 普通游戏窗口。黑底、洋红错误材质、静态贴图和颜色键只属于诊断失败，不得冒充正式身体就绪。

### Native 底座

`resources/native/<platform-arch>/` 只放可复用的平台原语，例如共享纹理、GPU 同步、窗口和 DPI 封装。它不包含 Avatar Shell 的业务监督逻辑；应用级 Composition Host 保持在 `components/avatar-shell/host/`。

## 用户数据目录

Windows 默认位置：

```text
%LOCALAPPDATA%/GlimmerCradle/
├── configs/                            # Control Center 写入的用户配置投影
└── data/
    ├── state/                          # 记忆、经历、数据库、扩展与外观状态
    ├── observability/                  # logs / metrics / traces / DLQ
    ├── blobs/                          # 图片、音频、附件等内容寻址对象
    ├── models/                         # 下载或导入的语音、视觉、向量与 Avatar 模型
    ├── packages/                       # 可重新安装的外部运行包
    ├── artifacts/                      # 可再生运行输出与临时媒体
    ├── cache/                          # 可清理索引和下载缓存
    ├── backup/                         # 迁移、升级和手动导出快照
    └── legacy/                         # 只读迁移源
```

月见自身的 Kernel、Cognition、Composition Host、Unity Worker 和 Engine 可执行文件不得进入用户数据目录。`data/packages/` 只用于用户环境中可重新安装的外部包。

## 写入与更新策略

| 区域 | 正常运行时可写 | 更新策略 |
|---|---|---|
| `resources/app.asar` | 否 | 随应用原子替换 |
| `resources/components/` | 否 | 按组件版本替换或修复 |
| `resources/native/` | 否 | 随平台组件更新 |
| `resources/assets/` | 否 | 随产品版本更新 |
| `resources/defaults/` | 否 | 新安装与迁移时作为默认输入 |
| 用户 `configs/` | 是 | 迁移，不无条件覆盖 |
| 用户 `data/state/` | 是 | 备份后迁移 |
| 用户 `data/models/`、`packages/` | 是 | 独立下载、校验与修复 |
| 用户 `data/cache/`、`artifacts/` | 是 | 可按策略清理重建 |

## 开发与安装映射

| 开发期 | 构建期 | 安装后 |
|---|---|---|
| `core/renderer/desktop/` | `out/release/desktop/` | `Selrena.exe` + `resources/app.asar` |
| `core/kernel/` | `out/release/kernel/` | `resources/components/kernel/` |
| `core/cognition/` | `out/release/cognition/` | `resources/components/cognition/` |
| `core/renderer/composition-host/` | `out/release/avatar-shell/host/` | `resources/components/avatar-shell/host/` |
| `core/renderer/avatar-unity/` | `out/release/avatar-shell/unity-worker/` | `resources/components/avatar-shell/unity-worker/` |
| `engines/` | `out/release/engines/` | `resources/components/engines/` |
| `native/` | `out/release/native/` | `resources/native/<platform-arch>/` |
| `assets/` | 资产导入投影 | `resources/assets/` |
| `configs/` 默认值 | 配置裁剪与校验 | `resources/defaults/` |
| `repo/data/` | 不装配 | `%LOCALAPPDATA%/GlimmerCradle/data/` |

`.deps/` 只作为构建输入；`out/` 可删除重建；`dist/` 保存安装器与便携发行物。三者都不成为用户连续性数据。

## 安装验收

- 首次启动不依赖仓库路径、开发工具或本机 Unity Editor。
- `component-manifest.json` 可解析所有必需入口和依赖。
- 安装目录在普通用户权限下保持只读，运行时不尝试写入。
- Unity Worker 启动时无普通窗口闪现，Avatar Shell 只显示带真实 alpha 的人物。
- 透明区可穿透，身体区域可交互，拖动不改变模型缩放或重建渲染器。
- 退出月见后没有 Kernel、Cognition、Engine、Host 或 Unity Worker 残留。
- 修复安装和版本升级不覆盖用户配置、记忆、模型与日志。
- 卸载时默认保留用户数据，并向用户提供明确的“同时删除本地数据”选择。
