# 月见 UI 设计指南

> 本文是历史 UI 设计材料。当前桌面/Avatar 技术边界见 [桌面与 Avatar 开发](../../guides/subsystems/桌面与Avatar开发.md)，长期分层见 [月见架构蓝图](../../architecture/blueprint/月见架构蓝图.md)。

## 1. 产品边界

月见的桌面体验由职责不同、语言一致的三个表面组成：

| 表面 | 核心职责 | 技术归属 |
|---|---|---|
| Control Center | 配置、状态、诊断和轻量对话 | Electron + React |
| Presence Surface | Unity 不可用时提供轻量状态与恢复入口 | Electron |
| Avatar Shell | Live2D / 未来 3D 身体、透明窗口和高频交互 | Unity |

Control Center 不是营销首页、模型控制台或完整聊天产品。它是月见的桌面管理舱：常用任务必须直接、状态必须可信、复杂细节必须按需展开。

## 2. 设计原则

Control Center 的基准风格称为**低装饰、内容优先的生产力工作台**。它不是某个框架皮肤，而是一组稳定判断：连续表面承载主要任务，色阶与留白形成软分区，工具和边界按需显现，交互状态比静态装饰更清楚。

1. **概念完整**：一个概念只有一个主要入口和一种状态语言；活动栏与侧边栏不得重复同一份页面菜单。
2. **任务优先**：首屏直接进入可操作工作区，不用 hero、欢迎广告或大面积装饰占据空间。
3. **渐进披露**：先给健康摘要和推荐动作，再提供详细参数、日志路径和原始错误。
4. **状态诚实**：区分连接中、准备中、可用、降级、失败和需重启，不用模糊的“在线”覆盖全部状态。
5. **交互可感知**：所有可点击、可拖动、可编辑元素都具备 hover、pressed、focus-visible、disabled 和 pending 反馈。
6. **密度适中**：配置和诊断可以紧凑，对话和空状态保留呼吸感；不使用巨型字号和移动端式大卡片堆叠。
7. **渲染器可替换**：Control Center 不因 PIXI、Cubism 或 Unity 的实现选择而改变信息架构。

## 3. 成熟产品借鉴

借鉴的是经过验证的结构能力，不复制品牌皮肤。

| 产品 | 借鉴内容 | 不照搬的部分 |
|---|---|---|
| VSCode | Workbench 的桌面密度、稳定工具栏、可调宽区域和状态反馈 | Activity Bar、上下文侧栏、命令面板等 IDE 复杂度 |
| Discord / Slack | 稳定导航、持续 Presence、消息与输入反馈、可辨识的未读和异常状态 | 娱乐化装饰、社交服务器层级和高噪声品牌色 |
| Telegram | 清晰的对话时间线、轻量气泡、发送与录音状态 | 把整个产品收缩为聊天窗口 |
| Codex Desktop | 主任务聚焦、克制留白、组合式输入区、上下文与动作靠近 | 首页式超大输入框和项目工作流语义 |
| Linear | 高密度列表、筛选、详情检查器、快速状态变更 | 面向团队工单的字段体系 |
| Windows Settings / Fluent 2 | 桌面设置分组、原生窗口行为、键盘和无障碍习惯 | 大面积半透明材质与系统应用外观复刻 |
| GitHub Desktop | 诊断入口、动作主次、错误恢复和明确的变更结果 | Git 专属双栏流程 |
| Obsidian | 记忆与知识的浏览、关联和局部检查器 | 无限画布作为默认导航 |
| Raycast | 清晰的动作命名和键盘可达性 | 全局命令面板、命令语法和快捷键优先的信息架构 |

## 4. Control Center 骨架

```text
Window Chrome
└── Workbench
    ├── Navigation Sidebar  # 品牌、分组页面、全局健康摘要
    ├── Main Workspace      # 当前任务的唯一主工作区
    ├── Inspector           # 仅在需要详情或保存检查时出现
    └── Feedback Layer      # toast、进度、错误与操作结果
```

推荐领域划分：

- **月见**：总览、对话。
- **管理**：配置、语音、形象。
- **认知**：记忆、知识与人格。
- **生态**：扩展与能力。
- **观察**：诊断、日志和启动状态。

当前页面数量和领域深度不足以支撑 IDE 式“活动栏 + 上下文侧栏”双层导航。Control Center 使用一条稳定的分组侧栏：领域作为分组标题，页面作为唯一可点击入口；侧栏底部保留 Kernel 与身体健康摘要。这样仍继承 Discord / VSCode 的稳定空间记忆，但不复制与任务规模不匹配的 IDE 层级。

侧栏允许在约束范围内调整宽度，并在窄窗口折叠为图标模式。桌面端双击品牌区可在完整与图标模式之间切换；不用只能收起、无法自然恢复的单向箭头占据导航空间。当前选中页由贴边的独立细指示条表示，它在页面切换与收展之间连续滑动；导航项本身不以位移或跳动制造点击反馈。侧栏只保留页面名称、图标和必要的领域标题，页面说明留给主工作区，避免导航成为第二份内容摘要。主工作区只保留一个主要滚动容器；Inspector 只在配置脏状态、诊断详情或对象详情需要时出现，禁止用永久右栏制造空白和卡片堆叠。

Control Center 不建设 IDE 式全局命令面板。页面只有 9 个左右，稳定导航比 `Ctrl+K` 更容易发现、更符合月见的使用频率。搜索必须有明确范围：配置页搜索配置项，记忆页搜索记忆，扩展页搜索扩展，诊断页筛选事件；不得把页面跳转、业务操作和任意命令混进同一个输入框。

## 5. 页面模式

### 总览

采用紧凑健康摘要，不使用营销式 hero。展示启动阶段、核心能力、当前外显、最近异常和下一步推荐动作；原始路径和完整错误进入诊断页。状态摘要优先使用连续表面、表格行和小型指标，不把每个数值做成独立大卡片。

### 对话

参考 Telegram 的时间线和 Codex 的组合输入区。对话以一块连续的会话表面承载：工具栏、正文与输入区使用轻边界、细微表面差和一致圆角形成包含关系，不把它们拆成堆叠卡片；输入区作为独立底座固定在时间线底部。正文保留 Markdown、代码块、复制与重试；`thought` 只显示短暂状态，不伪装成聊天消息。语音录制、识别、生成和播放必须分别反馈。

### 配置

参考 Windows Settings 的分组和 Linear 的检查器。配置来自受控投影，左侧主列按语义分组编辑，保存状态以粘性动作条或按需 Inspector 呈现；页面统一管理脏状态、校验、保存、重置和“需要重启”，密钥不以明文回显。禁止把表单沉到页面下方、让状态卡占据首屏主体。

### 语音与形象

同时展示目标 provider、实际 provider、资源准备、设备、最近失败和可执行修复。形象页只管理模型、显示比例、Avatar Shell 状态及预览，不在 Control Center 内嵌 Unity 身体。

形象页必须把 Avatar Shell 当成一等运行对象展示：当前外壳、模型、显示比例、Unity 资产投影、受管构建产物、runtime 包、进程日志和下一步修复动作应在同一个工作区可见。React 页面不直接读取文件系统或探测 Unity 进程；这些本机事实由 Electron main 收口成受控诊断投影，再通过 preload 暴露给 renderer。

形象页展示 runtime 状态时要区分模型格式与渲染实现：Cubism4/5 是模型兼容代际，官方 Cubism SDK for Unity 是实际渲染依赖。缺少 SDK 时显示为“需处理”，并给出安装目录或环境变量；不要把它描述成模型错误，也不要自动切换到 Q 版或 Web Live2D 冒充正式身体。

### 记忆与扩展

记忆页采用列表、搜索、关联详情的工作台；扩展页采用目录、贡献点、权限与健康状态。二者都避免仅展示占位说明。

### 诊断

先展示可读摘要，再提供日志和资源目录入口。错误卡必须包含影响、原因、推荐动作和技术详情展开项，不能只给红色状态词。

## 6. 视觉系统

- **主题优先**：Control Center 当前只维护自然的 `light / dark` 两套基础主题，并跟随系统切换。Light 的窗口和主工作区以接近纯白的中性表面为基底，避免把普通灰色大面积铺满而显得浑浊。先把桌面软件的明暗层级、可读性和控件状态做好，不叠加角色主题或动态取色。
- **色彩**：Light 使用中性白与浅灰表面，Dark 使用中性炭黑与深灰表面；两者统一使用系统蓝表达选中、焦点与主要动作。错误、警告和成功使用独立语义色。具体 token 见 [UI色彩参考](./UI色彩参考.md)。
- **材质**：采用 Fluent 2 的分层思想。窗口导航底层可使用低对比 Mica 式环境色，浮动菜单、对话框和临时 Inspector 可使用轻度 Acrylic 式半透明与模糊；主工作区、表单和长文本保持近乎不透明，确保稳定可读。材质表达窗口层级，不承担装饰职责，禁止整页和每张卡同时毛玻璃。
- **排版**：正文默认 13~15px，标题形成有限层级；桌面工作区不使用 viewport 缩放字号，字距保持 `0`。
- **间距**：以 4px 为基础单位，常用间距为 8 / 12 / 16 / 24px；同级元素共享节奏。
- **圆角**：普通控件 4~8px；卡片不超过 8px，除非明确采用 MD3 大容器。禁止卡片套卡片。
- **图标**：统一使用 Lucide 或既有图标集。常见操作优先图标，陌生图标必须有 tooltip 和无障碍名称。
- **层级**：优先通过表面色、边界和间距建立层次，阴影只用于弹层、浮动输入和必要的焦点提升。
- **连续表面**：导航、工具栏和工作区使用稳定底色组成完整窗口；卡片只用于可重复对象、独立状态组或浮层，不把整个页面切成漂浮方块。
- **软分区**：优先使用轻微明度差、对齐轴和留白划分区域。静态分割线保持低对比，只在窗口边界、连续列表和必要的结构断点出现；按钮与列表项默认低装饰，边界主要在 hover、focus、selected、error 时显现。

成熟产品参考及取舍记录见 [UI 参考索引](./ui-references/README.md)。

### 按钮系统

按钮按行为语义分为四类，不按页面临时发明样式：

| 类型 | 用途 | 外观 |
|---|---|---|
| Primary | 当前工作区唯一首要提交，如发送、保存、确认 | 实色强调，文字高对比 |
| Secondary | 普通页面动作，如重置、测试、打开详情 | 中性表面 + 细边界 |
| Ghost / Icon | 工具栏、窗口控制、低频局部动作 | 默认透明，hover 才出现表面 |
| Danger | 退出、删除、清空等不可逆动作 | 仅在危险语义下使用错误色 |

- 桌面按钮常规高度为 32px，重要提交可用 36px；不使用占满卡片宽度的巨型胶囊，除非它确实是移动端或单一确认流程。
- 圆角使用 6~8px，图标按钮保持稳定方形命中区；文字按钮使用图标时保持统一的 16px 图标尺寸和 8px 间距。
- `hover` 提升表面或边界，`pressed` 通过色阶收束与最多 1px 位移反馈，`pending` 保持原宽度并显示进度，避免界面跳动。
- 同一作用域只允许一个 Primary；保存未变更时必须 disabled，并说明原因或保持可理解的状态文案。

## 7. 交互状态

| 状态 | 必须表现 |
|---|---|
| Hover | 颜色或边界变化，提示可操作性 |
| Pressed | 轻微压缩、色阶变化或阴影收束，持续 80~140ms |
| Focus | 清晰的 `focus-visible` 焦点环，不能只依赖颜色 |
| Pending | 控件锁定、进度和可读状态，禁止重复提交 |
| Success | 就地确认结果，不让用户猜保存是否完成 |
| Error | 原因、影响、恢复动作；技术详情可展开 |
| Disabled | 降低强调并保留原因说明，不只是变灰 |

动效只解释状态变化，通常控制在 120~220ms；不得用持续漂浮、装饰性旋转或大面积渐变制造“活跃感”。

## 8. 响应式与可访问性

- 设计基线覆盖 `1280×720`、`1440×900` 和 `1920×1080`；窗口缩小时先收起 Inspector，再把分组侧栏折叠为图标模式。
- 所有主要流程支持键盘完成；可调分隔条提供 `role="separator"`、方向和数值范围。
- 文本对比度遵循 WCAG AA；状态不得只靠颜色表达。
- 输入框具有真实 label、错误关联和提交语义；图标按钮具有可读名称。
- Windows 缩放 125% / 150% 下不得重叠、裁切或产生不可达控件。

## 9. Playwright 验收

Playwright 是 renderer 的结构与交互回归层。浏览器测试通过 `selrenaAPI` mock 隔离 Kernel、模型下载、Unity 和原生窗口，Electron 专属生命周期另做集成验证。

```powershell
pnpm --filter @glimmer-cradle/desktop-surfaces test:ui:install
pnpm test:ui
pnpm --filter @glimmer-cradle/desktop-surfaces test:ui:report
```

最低覆盖：

1. 页面导航、窗口尺寸和主工作区不溢出。
2. 侧边栏拖动、折叠和键盘可达性。
3. 分组导航只有一份页面入口，领域标题不制造第二套导航。
4. 配置加载、校验、脏状态、保存、失败和重启提示。
5. 对话发送、录音状态和消息正文保真。
6. online / preparing / degraded / failed 等关键状态投影。
7. 稳定页面再建立视觉快照；动画、时间和动态数据必须先归一化。

测试产物统一写入 `data/artifacts/tests/playwright/`，不进入源码目录，也不提交报告、trace 或失败截图。

## 10. 禁止模式

- 为少量页面同时设置活动栏与上下文侧栏，制造重复导航和大面积空白。
- 为少量稳定页面引入全局命令面板、命令语法或快捷键体系。
- 首页巨型口号、营销文案或没有操作价值的大卡片。
- 整页由同尺寸卡片铺满，或卡片内部继续嵌套卡片。
- 只有颜色变化、没有 pressed / pending / focus 的“PPT 式按钮”。
- UI 直接读取 YAML、日志文件、Node API 或 Unity 进程。
- 把开发日志、绝对路径和原始 JSON 默认暴露在总览。
- 为一次故障增加永久 fallback、重复状态源或临时页面模型。
