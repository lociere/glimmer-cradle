# 日志字段约定（Log Fields Glossary）

> 迁移材料：当前规则见 `docs/reference/observability.md`。本页不再更新。

> 物理位置：跨进程跨语言的日志字段命名规范
>
> 定位：**日志结构的协议层**——任何写入 `runtime.log` / `runtime-error.log`
> 的字段名必须遵守此处约定，否则下游 grep / jq / Loki 查询会分裂。

> 文档状态：v1.0 / 2026-05-08 / 跟随 [架构 08](./08-记忆与日志架构.md) §3.3.7
> 落地。本文档与代码不一致时，**先改代码再改文档**（活文档原则）。

---

## 一、命名约定

- **全部 snake_case**（与 Python 自然风格 + 协议层契约对齐）
- **不允许 camelCase / PascalCase / kebab-case**——单一规则减少认知负担
- **保留字段不可重新含义**——如下表 §二 列的字段名仅用于其规定语义

## 二、保留字段（reserved fields）

由 logger 框架自动注入或调用方约定使用，**含义固定，不可改写**：

| 字段 | 类型 | 来源 | 含义 | 示例 |
|---|---|---|---|---|
| `level` | string enum | logger 自动 | 日志级别 `debug \| info \| warn \| error \| critical` | `"warn"` |
| `message` | string | 调用方 | 主消息文本（人类可读） | `"应用启动完成"` |
| `timestamp` | string (ISO 8601 ms) | logger 自动 | UTC 时间戳，毫秒精度 | `"2026-05-08T08:39:21.846Z"` |
| `module` | string | logger 自动 | 模块名（`getLogger(name)` 的 name） | `"app-root"` `"Extension:napcat-adapter"` |
| `source` | string | Kernel 内核 IPC 注入 | 仅出现在跨语言日志：`"cognition"` 标识源自 Python | `"cognition"` |
| `child_module` | string | Kernel 子进程日志转写 | 被监管子进程内部模块名；不得覆盖 Kernel 主日志的 `module` | `"self_entity"` |
| `child_event` | string | Kernel 子进程日志转写 | 被监管子进程原始事件名；用于摘要 warn/error | `"月见 已醒来"` |
| `trace_id` | string | logger 自动 | 一次跨层因果关联；缺失时合成 `run-{boot_id}-{n}` / `synthetic-{module}-{n}` | `"3c2a082a..."` |
| `boot_id` | string | logger 自动 | 一次进程启动周期；**仅认知核日志**（参考 systemd `boot_id`） | `"4074fad2..."` |
| `span_id` | string | logger 自动 | trace 内一个原子操作；存在才出现 | `"a1b2c3d4..."` |
| `error` | string | 调用方约定 | 错误对象的 `.message` 字符串化 | `"connect ECONNREFUSED 127.0.0.1:8001"` |
| `stack` | string | logger / 调用方 | 错误堆栈，多行字符串 | `"Error: ...\n    at ..."` |
| `critical` | boolean (always true) | logger 自动 | `logger.critical()` 注入；用于告警系统识别 | `true` |
| `exception` | object | structlog 自动 | Python 端异常详情（`ExceptionRenderer`） | `{...}` |

## 三、约定字段（convention fields）

业务可选写入，但**已有命名习惯**——尽量沿用以利于跨服务检索：

| 字段 | 类型 | 含义 | 适用场景 |
|---|---|---|---|
| `extension_id` | string | 扩展唯一 ID（`<bucket>/<id>` 或 `<id>`） | 扩展宿主、扩展业务日志 |
| `scene_id` | string | 场景 ID 原始形式（不 sanitize） | 记忆、事件、感知 |
| `event_type` | string | 事件类型（如 `"PerceptionEvent"`） | 事件总线、IPC |
| `subagent_id` | string | SubAgent 标识 | Agent / MCP 路由 |
| `tool_name` | string | MCP 工具名 | Tool 调用 |
| `attempts` | number | 重试次数 | 重连、退避 |
| `port` | number | 端口号 | 网络服务 |
| `path` | string | 文件路径 | I/O 操作 |
| `runtime_module` | string | 被生命周期编排器启动或停止的 runtime delegate 名称；不覆盖日志说话者 `module` | Kernel 启动/停机编排 |
| `startup_time_ms` | number | 启动耗时（ms） | 模块生命周期 |
| `total_startup_time_ms` | number | 应用总启动耗时（ms） | app-root 启动完成事件 |
| `code` | number | 退出码 / WebSocket close code | 进程退出、socket 关闭 |
| `signal` | string | POSIX 信号名 | 进程退出 |

## 四、不要使用的字段名（reserved by framework / risky）

| 字段 | 原因 |
|---|---|
| `name` | structlog `add_logger_name` 内部字段；与 `module` 同义但混用导致解析分裂 |
| `time` | 与 `timestamp` 同义；歧义 |
| `msg` | winston 内部别名 `message`；混用混淆 |
| `severity` | 与 `level` 同义；某些日志收集器只认其一 |
| `thread` | structlog 不自动注入，且 Python asyncio 没有线程语义；用 `trace_id` 代替 |
| `logger` | structlog 内部；调用方不应改写 |

## 五、字段值约定

### 5.1 时间戳

- **格式**：`YYYY-MM-DDTHH:mm:ss.SSS[Z]`（ISO 8601，毫秒精度，UTC）
- Python：`structlog.processors.TimeStamper(fmt="iso")` + `_normalize_timestamp_precision` 截断
- TS：`new Date().toISOString()`（已是 ms）

### 5.2 level 名称

| 协议规定 | Python stdlib | TS winston | 说明 |
|---|---|---|---|
| `debug` | `debug` | `debug` | 开发调试 |
| `info` | `info` | `info` | 正常运行信息 |
| `warn` | `warning` → 自动改写为 `warn` | `warn` | 警告（不阻断业务） |
| `error` | `error` | `error` | 异常（影响某次操作但不致命） |
| `critical` | `critical` | `error` + `critical: true` | 致命异常 / 即将退出 |

### 5.3 module 命名

- **Kernel 内核**：`kebab-case`（如 `app-root`、`config-manager`、`ipc-server`、`life-clock-manager`）
- **TS 扩展**：`Extension:<id>` 前缀（如 `Extension:napcat-adapter`）
- **Cognition 认知核**：`snake_case`（如 `cognition-manager`、`embedding_engine`、`long_term_memory`）
- **跨语言主线**：Kernel 主日志中的 `module` 永远表示 Kernel 侧说话者；Python / sidecar 等子进程的内部模块不得覆盖主线 `module`，
  应写入 `child_module`，完整内部时间线进入 `data/observability/logs/processes/`

### 5.4 trace 上下文规则（三层）

> **2026-05-19 重构（阶段 3.6）**：原四层 `session_id`/`epoch_id`/`trace_id`/`span_id`
> 收敛为三层 —— `session`/`epoch` 是认知主体（月见）的"清醒/心境"语义，已迁回
> **经历层**（`Moment` 自洽表达 + `causation_ids`），不再属于 telemetry。

telemetry 层 trace 共三层（月见架构蓝图 §6.2，对齐 W3C TraceContext / OTel）：

| 层 | 粒度 | 范围 |
|---|---|---|
| `boot_id` | 一次进程启动周期 | 认知核进程级（参考 systemd `boot_id`） |
| `trace_id` | 一次跨层因果关联 | 协程级，跨进程透传 |
| `span_id` | trace 内一个原子操作 | 协程级，跨进程透传 |

- **trace_id 正常值**：UUIDv4 hex（无连字符），如 `"3c2a082adcba4d3289..."`
- **trace_id 合成**（无上游入口）：`boot_id` 已确立 → `run-{boot_id}-{n}`（与启动
  周期绑定，跨 run 不撞）；`boot_id` 尚未确立（启动极早期）→ `synthetic-{module}-{n}`
- **不允许**：空字符串、`null`、`undefined`——logger 自动 fallback 为合成值
- **生命周期**：`boot_id` 在进程启动时设定（认知核 `main.py`）；`trace_id` / `span_id`
  在事件入口通过 `withTrace(id, fn)`（TS）/ `with TraceContext(id):`（Python）建立，
  其内部所有日志、异步任务自动继承
- **内核侧**：Kernel 内核是基础设施，无 `boot_id` —— 内核日志只注 `trace_id` + `span_id`
- **经历事件上**：`trace_id` 是薄注解（可空），真正的因果链由 Moment 的
  `causation_ids` 表达（见 [08-记忆与日志架构](./08-记忆与日志架构.md) §8.3）

## 六、变更流程

修改本字段表 = 修改协议层。流程：

1. PR 文档（本表）+ 涉及 logger / event handler 的代码改动一并提交
2. CI 检查：grep 仓库内 logger 调用，反对样违反字段约定的提交
3. 现有日志数据兼容：旧 logfile 不需要迁移；解析工具按"字段缺失即跳过"宽容处理

## 六bis、metrics sink（独立于 runtime.log）

遥测支柱 ② metrics 是**独立 sink** —— `data/observability/metrics/metrics-{proc}.jsonl`，
不写入 `runtime.log`。每条事件字段：`ts` / `name` / `kind`（counter|gauge|histogram）/
`value` / `labels` / `trace_id` / `boot_id`。字段同样遵守 snake_case
约定；详见 [阶段3-遥测设计](../../history/architecture-decisions/阶段3-遥测设计.md) §四②。

## 六ter、traces / span sink（独立于 runtime.log）

遥测支柱 ③ span（trace 内一个原子操作）也是**独立 sink** ——
`data/observability/traces/spans-{proc}.jsonl`，不写入 `runtime.log`。每条事件字段
（结构对齐 OpenTelemetry Span 精简版）：

| 字段 | 类型 | 含义 |
|---|---|---|
| `name` | string | span 名（如 `"llm.generate"` / `"memory.retrieve"`） |
| `trace_id` | string | 所属 trace（无上游时 span 入口现场新建） |
| `span_id` | string | 本 span 唯一 ID（16 hex chars，对齐 OTel） |
| `parent_span_id` | string \| null | 父 span；根 span 为 `null` |
| `started_at` / `ended_at` | string ISO ms | 起止时间戳 |
| `duration_ms` | number | 耗时（毫秒，浮点） |
| `status` | string enum | `"ok"` \| `"error"`；异常路径自动标 error |
| `error` | string \| null | 异常类型名（status=error 时） |
| `attributes` | object | 自定义属性（如 `model`/`scene_id`/`tokens`） |
| `boot_id` | string \| null | 进程启动周期（仅认知核） |

跨进程父 span：调用方在 IPC 信封透传 `trace_id` + `span_id`，被调方入站时
通过 `with_remote_parent_span`（Python） / `withSpan`（TS）建为父上下文，
子 span 自动挂到远端父之下，形成跨进程 span 树。详见
[阶段3-遥测设计](../../history/architecture-decisions/阶段3-遥测设计.md) §四③。
