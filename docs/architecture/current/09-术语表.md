# 术语表

> 范围：跨文档、跨语言和跨进程使用的核心术语；不作为字段或 API 的权威表。
> 事实依据：架构蓝图、Current、Implementation、Protocol Schema、Extension SDK 与 Reference。
> 维护触发：新增公开概念、重命名、边界改变或术语在多个文档中出现冲突。

| 术语 | 含义 | 常见误用 |
|---|---|---|
| Glimmer Cradle / 微光摇篮 | 本地优先数字生命体企划、运行平台与仓库整体 | 把企划平台等同于某一个角色 |
| Selrena / 月见 | 当前默认角色、主线人格实例和默认身体资产归属 | 把 Selrena 当成平台、协议、Kernel 或 SDK 的通用命名 |
| Character Profile | 一个可激活角色的身份、人格、知识、唤醒词、声音、Avatar 资产和数据命名空间组合 | 为每个角色复制一套 Kernel、Protocol 或 Extension SDK |
| Blueprint | 架构宪法、设计审美、长期不变量和目标形态 | 当成过期 vision 或路线图 |
| Current Architecture | 当前系统结构、边界、运行时关系和质量要求 | 写字段表或操作步骤 |
| Implementation | 当前代码如何实现架构概念的入口、组装、链路和调试地图 | 只写原则，不指向代码 |
| Reference | 字段、配置键、目录、SDK、日志和打包映射的精确事实 | 写背景故事或开发流程 |
| Guide | 完成某类任务的步骤、失败分支、验证和文档同步 | 复制架构正文 |
| Kernel | 生命周期、路由、Ingress Gate、能力编排、状态投影和权限中枢 | 人格核、回复生成器 |
| Cognition | Python 认知核，负责人格、情绪、经历、记忆、推理、巩固和行动语义 | 平台 IO 层、窗口控制层 |
| Character | 当前激活人物的完整本体，拥有身份、认知连续性、声音与 Avatar | 把某个模型或进程当成完整人物 |
| Character Package | 作者维护的身份种子、人格、对话与安全配置 | 被运行期记忆静默改写的 persona 数据库 |
| Experience Ledger | 由全局有序不可变 Moment 构成的角色经历事实源 | 普通日志、transcript 或聊天历史 |
| Moment | Ledger 的最小经历单元，包含来源、因果、场景、参与者和保留上限 | 任意 telemetry event |
| SourceDescriptor | 描述信息如何进入 Cognition 的 provider、事件、schema、trust 与 privacy | 只有自由文本的 source 字段 |
| Episode | 从 Ledger 派生、可重建的经历边界与巩固批次 | 第二份经历事实或单个文件 |
| Consolidation | 从封口 Episode 生成并校验有证据认知修订的过程 | 无证据的 LLM 总结或 mock 反思 |
| Memory Substrate | 版本化 Memory、revision、evidence、relation、relationship 与 intention 状态层 | 单表长期记忆或 Memory Graph 事实源 |
| Evidence Proposal | Extension 提交的带来源候选，必须先成为 Moment 再由 Cognition 判断 | Extension Memory 写权限 |
| Avatar | Character 拥有的形象与身体领域，包含 Package、状态、动作和行为 | Surface、Unity 进程或可选外挂 |
| Avatar Host | 承载某种 Avatar 实现的进程边界 | Avatar 本体或 Surface |
| UnityAvatarHost | 当前 Unity/Cubism Avatar Host 实现 | 通用 Avatar 领域名 |
| Surface | 人物出现和用户交互发生的位置 | Avatar 本体或系统事实源 |
| Renderer / Desktop Surface | Electron/React 桌面 Surface，包括 Control Center 与 Presence | 系统事实源或配置编辑器后端 |
| Control Center | 桌面控制中心，配置、状态、诊断和轻量对话入口 | 正式身体或完整聊天产品 |
| Presence | Avatar 不可用或让位时的轻量存在入口 | Unity 失败后的假 ready Avatar |
| Composition Host | Native 桌面合成边界，负责 alpha、透明命中、拖动、DPI、多显示器 | Unity 模型逻辑或 UI 容器 |
| Engine | 官方拥有的模型/媒体能力执行器，如 audio | Extension 或外部 MCP server |
| Extension | 可安装、授权、禁用、升级和回收的生态能力单元 | Kernel 内部模块或本体器官 |
| Skill Plane | 统一管理 Core/Extension/MCP/User 能力的运行平面 | Codex IDE skill 或普通插件列表 |
| Skill Provider | 提供 skill/tool/resource/prompt 的来源 | 只指 Extension |
| MCP Server | 外部工具/资源/Prompt 服务，通过 MCP Provider 接入 Skill Plane | 可信 Kernel 内部服务 |
| Protocol | 跨语言、跨进程和公开 SDK 稳定契约及生成投影 | 任意共享 TS 类型 |
| Port | 跨边界公开能力接口，由 owner 暴露给消费者 | 内部 service 实例 |
| Projection | 面向 UI、日志、catalog 或诊断的只读投影 | 可回写事实源 |
| Fact Source | 某类状态的唯一权威来源 | 多处“互相同步”的副本 |
| Readiness | 能力满足真实业务服务条件 | 进程存在或端口连接 |
| Degraded | 能力未完整可用但系统能解释并继续部分服务 | 静默 fallback |
| Ingress Gate | Kernel 控制用户/平台输入是否可进入认知链路的入口闸门 | UI 是否显示的判断 |
| Trace | 关联一次跨模块操作的上下文 ID 与 span 链路 | 单条日志 ID |
| DLQ | 无法安全处理但需要追查的失败事件存储 | 普通错误日志 |
| State | 不可随意丢弃的用户或系统连续性 | 缓存 |
| Cache | 可删除、可重建的性能材料 | 记忆或迁移源 |
| Artifact | 运行或构建过程中生成的产物 | 源码事实源 |

新增术语时先判断它是否已经能由现有术语表达。若只是某个子系统内部实现名，不应进入全局术语表；若跨文档、跨语言或对外 API 会使用，则必须同步蓝图、Current、Reference 或 SDK 命名。
