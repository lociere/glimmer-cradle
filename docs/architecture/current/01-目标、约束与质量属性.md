# 目标、约束与质量属性

> 范围：Glimmer Cradle 当前系统必须满足的产品目标、架构约束和质量属性；不列具体字段、配置值或阶段计划。
> 事实依据：根目录 `README.md`、[架构蓝图](../blueprint/微光摇篮架构蓝图.md)、workspace 结构、运行时实现与项目协作约定。
> 维护触发：产品定位、支持平台、核心运行时、进程边界、安全基线或质量目标变化。

Glimmer Cradle 是一个本地优先的数字生命体企划与运行平台。当前默认角色是 Selrena（月见），她具有连续人格、经历、记忆、情绪、主动认知、行动能力和桌面身体。当前架构不是围绕“聊天窗口”组织，而是围绕“一个能在本机长期驻留、可感知、可表达、可降级、可解释，并可承载角色 profile 的生命体运行时”组织。

## 核心目标

| 目标 | 当前架构含义 | 不接受的做法 |
|---|---|---|
| 连续人格 | Cognition 拥有当前激活角色的人格、情绪、记忆、经历与行动语义 | Kernel、Renderer 或 Extension 代替 Cognition 判断“角色应该怎么想” |
| 本地优先 | 默认数据、模型、日志和可观测材料落在本机可控域 | 把用户状态隐式写入第三方服务或安装目录 |
| 多进程隔离 | Kernel、Cognition、Desktop、Engine、Avatar、Extension/MCP 有清晰故障域 | 一个进程同时承担 UI、模型、平台 IO、生命周期和人格判断 |
| Avatar 本体 | Avatar 是 Character 的形象与身体；Presence 和 Desktop 是 Surface | Host 失败后用 Presence 冒充 Avatar ready |
| 能力生态 | Extension、MCP、Core Skill、User Skill 通过 Skill Plane 统一发现、授权、调用和审计 | 扩展 import Kernel 内部对象或绕过 Policy 直接调用工具 |
| 可演进协议 | 跨语言、跨进程和 SDK 结构由 Schema/公开接口演进 | 手写镜像类型、UI model 反向成为协议或永久双轨 |

## 架构硬约束

- **Kernel 是中枢，不是人格核。** Kernel 管生命周期、路由、能力编排、状态投影、权限和诊断；不决定回复内容、人格变化或记忆语义。
- **Cognition 是认知核，不是平台进程。** Cognition 管人格、情绪、经历、记忆、推理、巩固、计划和行动语义；不直接打开窗口、读桌面 API、管理扩展进程或触碰外部平台原始 payload。
- **Renderer 是 Surface，不是事实源。** Control Center 与 Presence 只消费 preload/Kernel 提供的受控投影；Avatar Host 同样只消费协议投影，不读取 Kernel 内部对象。
- **Extension 是可选生态能力，不是本体器官。** 可安装、禁用、授权、升级和回收的集成进入 Extension/MCP；音频、Avatar、Cognition、Kernel 等本体能力不伪装为 Extension。
- **Protocol 是跨边界契约，不是便利类型库。** 两种语言或两个进程共同消费的结构进入 `protocol/src/schemas/`；生成物只读。
- **readiness 必须真实。** 进程存活、端口连接、协议握手、资源准备、首帧呈现和业务可用是不同状态；不得用 fallback 冒充 ready。

| 属性 | 判定方式 | 相关文档 |
|---|---|---|
| 边界清晰 | 每个概念有唯一 owner；跨边界只走 Protocol、Port 或受控投影 | [构件、分层与依赖](./04-构件、分层与依赖.md) |

## 平台与工具约束

- TypeScript workspace 使用 `pnpm`；Python 项目使用 `uv`。不得用全局 `pip` 或手工拷贝依赖替代项目环境。
- Windows 是桌面发行目标；路径、窗口、透明合成、DPI、多显示器和录屏约束必须以真实 Windows 行为验证。
- 密钥、token、provider key 只允许进入 `configs/secrets/` 或环境变量；不得进入 Git、日志、文档、Skill、示例或 agent profile。
- 源码、只读资产、用户数据、模型缓存、第三方包、构建产物和运行缓存必须按 [data-layout](../../reference/data-layout.md) 分域。

当以上目标与当前实现冲突时，优先修正实现或记录 ADR；不能用文档绕过蓝图不变量。蓝图中的“优雅、完整、无多余边角”在 Current 中落实为：每个模块都能从目标、边界、生命周期和验证中说明其存在理由。
