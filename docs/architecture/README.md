# Architecture

Glimmer Cradle 的架构文档有三层，必须按问题选择阅读入口。

| 层 | 回答的问题 | 入口 |
|---|---|---|
| 蓝图 | 企划平台与当前默认角色为何如此设计、应守住什么、最终要长成什么 | [blueprint/](./blueprint/README.md) |
| 当前架构 | 当前系统由什么组成、如何运行、边界在哪里 | [current/](./current/README.md) |
| 技术实现 | 这些边界在真实代码中如何组装、通信、启动和验证 | [implementation/](./implementation/README.md) |
| 架构决策 | 为什么作出长期且影响边界的取舍 | [decisions/](./decisions/README.md) |

蓝图是 Glimmer Cradle 的架构宪法，不是旧材料；当前架构是代码事实的系统视图；实现文档是从架构概念到源代码的解释地图。精确字段查 [Reference](../reference/README.md)，操作步骤查 [Guides](../guides/README.md)。
