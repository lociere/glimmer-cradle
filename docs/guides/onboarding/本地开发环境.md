# 本地开发环境

> 适用场景：首次在本机配置 Glimmer Cradle 开发环境，或排查依赖/启动环境问题。
> 前置条件：Windows 开发机；已安装 Git、Node.js、pnpm、Python/uv；不要用全局 pip 替代项目环境。

## 初始化

```powershell
pnpm install
```

Python 子项目使用各自 uv 环境：

```powershell
cd core/cognition
uv sync --extra dev

cd ../../engines/audio
uv sync --extra dev
```

需要完整语音能力时：

```powershell
cd engines/audio
uv sync --extra tts --extra asr --extra dev
```

## 配置

- 普通配置在 `configs/system/`、`configs/characters/<character-id>/`、`configs/extensions/`。
- 密钥只放 `configs/secrets/` 或环境变量。
- 不把 provider key、token、私人模型路径写入文档、示例或日志。
- Avatar/音频/模型路径通过 resolver 和 catalog 处理，不在代码里写本机绝对路径。

## 常用启动与验证

```powershell
pnpm typecheck
pnpm build
pnpm dev
```

三条命令属于不同阶段：`pnpm build` 是发布/验收构建；`pnpm prepare:runtime` 只按内容指纹准备 Protocol、Extension SDK、已启用本地扩展和 Desktop 发布资产；`pnpm dev` 会先执行同一增量预检，再启动 Kernel、Cognition、Desktop 与受管 sidecar。源码和配置未变化时，启动不会重复执行全仓 build。直接运行某个 workspace 子包前，应先在仓库根执行一次 `pnpm prepare:runtime`。

Cognition：

```powershell
cd core/cognition
uv run pytest -q
```

Audio：

```powershell
cd engines/audio
uv run pytest -q
```

UI：

```powershell
pnpm test:ui
```

## 常见问题

| 症状 | 处理 |
|---|---|
| Vite 端口被占用 | 释放固定端口；开发期不应自动漂移到随机端口 |
| Kernel 未 ready 但 Electron 已显示 | 这是等待态；查看 Kernel readiness 和 process logs |
| Python 包找不到 | 在对应子项目运行 `uv sync`，不要全局 `pip install` |
| 模型下载到用户全局缓存 | 检查 catalog/cache 配置，应落在 `data/models/` |
| Avatar 缺 SDK | 运行 avatar doctor/sync/build，并查看 process log |
| TTS/ASR 不可用 | 先跑 audio `health` 和 warmup，确认资源 catalog |

## 验收

本地环境完成的最低标准：`pnpm typecheck` 可运行；Cognition/Audio 对应测试可运行；`pnpm dev` 能展示 Desktop 等待/ready/degraded 状态；日志写入 `data/observability/`；密钥未进入 Git。
