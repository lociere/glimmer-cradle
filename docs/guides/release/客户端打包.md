# 客户端打包

> 适用场景：构建 Glimmer Cradle 桌面客户端、验证安装投影、检查 Avatar/Audio/native 等组件是否可发布。
> 前置条件：已读 [Packaging Layout Reference](../../reference/packaging-layout.md)，工作树不包含无关本地资产或密钥。

## 步骤

1. 执行 `git status --short`，确认不会把私人模型、secret、缓存或测试输出打入包。
2. 运行基础验证：

```powershell
pnpm typecheck
pnpm build
```

3. 如涉及 Avatar，运行 avatar 同步、doctor、build 流程，确认 Unity SDK、模型 catalog、native composition 组件和 Host executable。
4. 如涉及 Audio，运行 audio 测试和必要的 `health`/warmup，确认模型目录与 resource catalog。
5. 构建客户端包。
6. 用打包产物启动，不用开发服务器结果代替发布验收。
7. 检查用户数据、日志、模型、缓存是否写入 user-data/data 域，而不是安装目录。

## 发布检查

| 检查 | 要求 |
|---|---|
| Secret | 包内无真实 `configs/secrets/secrets.yaml`、provider key、token；只允许空值 `secrets.example.yaml` 模板 |
| 私有资产 | 包内无私人 Avatar 模型、训练语料、个人声线，除非明确发布资源 |
| required 组件 | Kernel、Cognition、Desktop、Audio/Avatar required 组件能定位 |
| ready/degraded | 缺资源时能解释 degraded，不伪装 ready |
| Avatar | 打包版验证透明、DPI、多显示器、拖动、动作、口型、退出 |
| Audio | TTS/ASR provider、cache、process log、timeout 可诊断 |
| Logs | 主日志和 process log 写入 `data/observability/` |
| 停机 | 退出能回收 sidecar、stdio、WebSocket 和子进程树 |

## 失败与恢复

- 缺 required 构建输入：停止打包，补资源或改为明确 optional。
- 打包版写安装目录：修 resolver，不通过改权限绕过。
- Avatar 只有 `host_hello` 没有 `host_ready`：查 Unity model/SDK/Composition Host 首帧，不把连接当 ready。
- Audio 首次请求才下载模型：把资源准备前移到 warmup/readiness。
- 包内出现 secret 或私人资产：删除包，修打包规则和 ignore，再重新构建。
